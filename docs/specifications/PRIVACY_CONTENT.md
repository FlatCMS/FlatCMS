## Règle

```text
Aucun champ entre crochets et aucune hypothèse non validée ne doivent
subsister dans la version publiée.
```

- le registre des traitements ;

# 10. Bases légales

## H2

```text
Sur quelles bases les traitements reposent-ils ?
```

Chaque traitement doit reposer sur une base déterminée avant son
activation.

## Contrat

Peut être utilisé lorsque le traitement est objectivement nécessaire :

- création d’un compte payant ;
- exécution d’un abonnement ;
- activation ;
- support inclus ;
- paiement ;
- livraison d’un composant.

## Mesures précontractuelles

Peuvent couvrir :

- demande de devis ;
- demande multi-domaines ;
- question commerciale préalable ;
- préparation d’une offre.

## Obligation légale

Peut couvrir :

- facturation ;
- comptabilité ;
- obligations fiscales ;
- réponse à une autorité habilitée ;
- conservation de certaines preuves.

## Intérêt légitime

Peut être envisagé pour :

- sécurité ;
- prévention de fraude ;
- amélioration technique ;
- réponse à certaines demandes ;
- prospection de clients professionnels dans certaines conditions.

Une mise en balance doit être documentée.

## Consentement

Peut être requis pour :

- newsletter ;
- cookies non nécessaires ;
- publicité ;
- certains usages IA optionnels ;
- certains transferts ou fonctions non indispensables.

Le retrait doit être aussi simple que l’accord.

## Ne pas faire

```text
Ne pas sélectionner « consentement » par défaut pour tout.
```

# 29. Durées de conservation

## H2

```text
Combien de temps les données sont-elles conservées ?
```

Les données ne doivent pas être conservées indéfiniment.

## Phases

- base active ;
- archivage intermédiaire ;
- suppression ou anonymisation ;
- archivage définitif uniquement si justifié.

## Tableau de travail

| Donnée / traitement | Base active proposée | Archivage à confirmer |
|---|---:|---:|
| Contact général | 12 mois après clôture | preuve limitée |
| Support | durée du contrat + 12 mois | prescription applicable |
| Compte actif | durée du compte | obligations et litiges |
| Compte inactif | notification puis suppression | preuve limitée |
| Licence | durée active | preuve contractuelle |
| Factures | selon obligation légale | généralement archivage légal |
| Newsletter | jusqu’au retrait | preuve du retrait |
| Prospects | durée limitée à confirmer | opposition |
| Logs web | quelques mois à confirmer | incident uniquement |
| Logs sécurité | durée adaptée au risque | incident et preuve |
| Consentement cookies | durée à confirmer | preuve limitée |
| Démo | très courte | aucune donnée réelle |
| Prompt IA | aucune ou durée minimale | selon fournisseur |
| Exercice des droits | durée de traitement | preuve |
| Signalement sécurité | durée de résolution | preuve et obligations |

## Important

Les durées finales doivent être :

- justifiées ;
- documentées ;
- automatisées ;
- cohérentes avec les sauvegardes ;
- indiquées dans le registre.

# 31. Violations de données

## H2

```text
Gestion des incidents
```

Une violation peut concerner :

- confidentialité ;
- intégrité ;
- disponibilité.

## Procédure interne

1. détecter ;
2. contenir ;
3. documenter ;
4. évaluer ;
5. corriger ;
6. notifier si nécessaire ;
7. informer les personnes si risque élevé ;
8. tirer les enseignements.

## CNIL

Une notification à la CNIL peut être requise dans les délais prévus par
le RGPD lorsque la violation est susceptible d’engendrer un risque.

## Registre

Toute violation doit être documentée, y compris lorsqu’elle n’est pas
notifiée.

# 32. Droits des personnes

## H2

```text
Quels sont vos droits ?
```

Selon le traitement, une personne peut disposer :

- du droit à l’information ;
- du droit d’accès ;
- du droit de rectification ;
- du droit à l’effacement ;
- du droit à la limitation ;
- du droit d’opposition ;
- du droit à la portabilité ;
- du droit de retirer son consentement ;
- du droit de ne pas faire l’objet de certaines décisions automatisées ;
- du droit d’introduire une réclamation.

## Limites

Ces droits ne sont pas absolus.

Une obligation légale ou la défense de droits peut justifier la
conservation de certaines données.
