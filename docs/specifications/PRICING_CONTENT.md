## 1. Objectif de la page

Cette page présente les tarifs des composants premium de FlatCMS.

Elle doit permettre de comprendre immédiatement :

- que FlatCMS LTS Core reste gratuit et open source ;
- que PagesBuilder, MenuBuilder et FooterBuilder sont des composants commerciaux ;
- que chaque offre fonctionne sous la forme d’un abonnement annuel ;
- qu’une licence couvre un nom de domaine de production ;
- que les Builders peuvent être utilisés gratuitement en environnement local ;
- qu’un test sur un domaine de production sans licence affiche un bandeau ;
- que l’expiration d’un abonnement ne coupe jamais le frontend ;
- que les contenus déjà construits restent affichés et conservés ;
- que les mises à jour, l’édition en production et le support dépendent d’une licence active.

La grille doit rester simple au lancement.

Elle ne doit pas proposer de fausses réductions, de compte à rebours artificiel ou d’offre Agence non encore définie.

```text
1 abonnement
```

## Exemples reconnus pour un même domaine

Selon la normalisation retenue par le système de licences :

```text
example.com
www.example.com
```

peuvent représenter le même domaine de production.

## Sous-domaines publics

Les sous-domaines publics distincts ne sont pas automatiquement couverts.

Exemples :


## Environnements locaux

Les environnements locaux sont utilisables gratuitement pour construire
et tester le site.

```text

site.local
```

Le nombre et le format des environnements de préproduction associés
doivent être confirmés avant publication définitive.

# 11. Que se passe-t-il à l’expiration ?

## H2

```text
Le frontend et les contenus restent en place
```

## Principe absolu

```text
FlatCMS ne coupe jamais le frontend à l’expiration d’une licence Builder.
```

Les pages, menus et footers déjà construits restent :

- affichés ;
- accessibles ;
- enregistrés ;
- sauvegardables ;
- restaurables ;
- indépendants de la disponibilité du serveur de licences.

## Aucune suppression

L’expiration ne doit jamais :

- supprimer une page ;
- retirer un widget ;
- masquer un menu ;
- désactiver un footer ;
- modifier le JSON enregistré ;
- effacer un layout ;
- provoquer une erreur HTTP 500 ;
- afficher une page blanche ;
- remplacer le contenu par un message de licence.

## Après expiration

Les fonctions suivantes peuvent être suspendues :

- édition en production ;
- publication de nouvelles modifications ;
- téléchargement des mises à jour ;
- accès aux nouvelles ressources premium ;
- support commercial ;
- transfert de la licence vers un nouveau domaine.

## Mode recommandé

```text
Éditeur en lecture seule
```

Le client peut :

- ouvrir le Builder ;
- consulter la structure ;
- prévisualiser ;
- sauvegarder ou exporter ;
- vérifier l’état de l’abonnement ;
- renouveler.

## Message d’administration

```text
Votre abonnement a expiré. Le site public et les contenus existants
continuent de fonctionner. Renouvelez la licence pour réactiver
l’édition, les mises à jour et le support.
```

## Aucun bandeau public d’expiration

```text
L’expiration d’un abonnement précédemment valide ne doit pas faire
apparaître un nouveau bandeau commercial sur le frontend public.
```

# 13. Mises à jour

## H2

```text
Des mises à jour pendant toute la durée de l’abonnement
```

Une licence active donne accès aux mises à jour du ou des Builders
souscrits pendant douze mois.

## Mises à jour possibles

- correctifs ;
- compatibilité avec FlatCMS LTS ;
- améliorations ;
- nouveaux widgets ;
- nouvelles options ;
- corrections de sécurité ;
- mises à jour des modèles ;
- évolution du système de licences.

## Limites

Une mise à jour peut nécessiter :

- une version minimale de FlatCMS ;
- une version PHP compatible ;
- une sauvegarde ;
- une migration de données ;
- un thème compatible ;
- une mise à jour intermédiaire.

## Bonne pratique

```text
Sauvegardez et testez toute mise à jour en local ou en préproduction avant
de l’appliquer au site public.
```

## Après expiration

Le site continue à fonctionner avec la dernière version installée, mais
les nouvelles mises à jour ne sont plus disponibles jusqu’au
renouvellement.

# 16. Changer de Builder ou d’offre

## H2

```text
Faire évoluer son abonnement
```

## Upgrade recommandé

Autoriser pendant la période active :

```text
Builder Solo
→ Bundle Duo
→ Suite complète
```

Le montant déjà payé peut être pris en compte au prorata de la période
restante si le système de facturation le permet.

## Exemple conceptuel

Un client ayant acheté PagesBuilder peut passer à la Suite complète en
payant la différence calculée pour la durée restante.

## Downgrade

Le passage à une offre inférieure peut prendre effet :

- immédiatement avec crédit ;
- ou au prochain renouvellement.

La règle doit être définie avant publication.

## Changement de sélection du Bundle Duo

À confirmer :

- changement libre au renouvellement ;
- changement payant en cours de période ;
- nombre maximal de changements ;
- conservation du rendu des Builders retirés.

## Règle de sécurité

Même lorsqu’un Builder est retiré de l’abonnement, les contenus déjà
construits doivent continuer à être rendus sur le frontend.

# 22. Questions fréquentes éditoriales

> Ces réponses n’impliquent pas automatiquement un balisage `FAQPage`.

## H2

```text
Questions fréquentes sur les tarifs
```

### H3 — FlatCMS est-il payant ?

```text
FlatCMS LTS Core reste gratuit et open source. Seuls les Builders et
certains composants commerciaux nécessitent une licence distincte.
```

### H3 — Puis-je tester un Builder gratuitement ?

```text
Oui. Les Builders peuvent être utilisés gratuitement dans un
environnement local afin de construire et tester le site.
```

### H3 — Une licence couvre-t-elle plusieurs sites ?

```text
Non. Au lancement, une licence couvre un nom de domaine de production
pendant douze mois.
```

### H3 — Puis-je utiliser `www.example.com` avec une licence pour `example.com` ?

```text
Le domaine nu et sa variante `www` doivent être reconnus comme le même
domaine après normalisation, selon la politique technique finale.
```

### H3 — Un sous-domaine est-il inclus ?

```text
Un sous-domaine public distinct peut constituer un site différent. Les
environnements locaux et de préproduction reconnus suivent une politique
spécifique.
```

### H3 — Que se passe-t-il si je teste un Builder en production sans licence ?

```text
Le contenu reste affiché, mais un bandeau indique le mode test jusqu’à
l’activation d’une licence. Les mises à jour et le support commercial ne
sont pas disponibles.
```

### H3 — Mon site disparaît-il si l’abonnement expire ?

```text
Non. Les pages, menus et footers déjà construits restent affichés.
FlatCMS ne coupe jamais le frontend et ne supprime pas les données
Builder à l’expiration.
```

### H3 — Un bandeau apparaît-il après expiration ?

```text
Non sur le frontend d’un abonnement précédemment valide. L’avertissement
d’expiration reste dans l’administration. Le bandeau public concerne
uniquement le mode de test en production sans licence.
```

### H3 — Puis-je encore modifier le site après expiration ?

```text
La politique recommandée place l’éditeur en lecture seule en production.
Le rendu et les données restent disponibles, mais le renouvellement est
nécessaire pour reprendre l’édition, les mises à jour et le support.
```

### H3 — Puis-je renouveler plus tard ?

```text
Oui. Le renouvellement réactive les fonctions commerciales sans supprimer
ni recréer les contenus déjà construits.
```

### H3 — Puis-je changer de domaine ?

```text
Oui selon la procédure de transfert définie. L’ancienne activation doit
être libérée avant l’activation du nouveau domaine.
```

### H3 — Les mises à jour sont-elles incluses ?

```text
Oui pendant les douze mois de l’abonnement actif pour les Builders
souscrits.
```

### H3 — Le support est-il inclus ?

```text
Oui, pour les questions techniques relevant des Builders et dans les
limites des conditions de support publiées.
```

### H3 — Existe-t-il une licence Agence ?

```text
Aucune offre Agence publique n’est encore définie. Les besoins
multi-domaines peuvent faire l’objet d’une demande spécifique.
```

## Informations d’une offre

- prix ;
- devise `EUR` ;
- disponibilité ;
- URL ;
- durée ou période d’abonnement visible ;
- vendeur ;
- taxes non incluses si cela correspond à l’affichage ;
- nom exact de l’offre.

## Prudence

Schema.org ne doit pas annoncer :

- une garantie non publiée ;
- une réduction expirée ;
- un avis fictif ;
- un prix TTC incorrect ;
- une offre Agence inexistante ;
- une disponibilité non réelle.
