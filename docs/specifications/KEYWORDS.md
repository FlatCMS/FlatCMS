> **Document directeur de visibilité**
>
> Projet : FlatCMS  
> Domaine cible : `https://flat-cms.fr`  
> Version de référence : FlatCMS v1.0.0 LTS Core  
> Date de création : 8 juin 2026  
> Documents parents : `SEO.md`, `SITE_ARCHITECTURE.md`, `CONTENT_MATRIX.md`, `DOCUMENTATION_MAP.md`  
> Statut : cartographie initiale à valider avec les données Search Console après lancement

---

## 1. Objet du document

Ce document définit la stratégie de mots-clés du futur site officiel FlatCMS.

Il organise :

- les requêtes de marque ;
- les requêtes génériques ;
- les requêtes techniques ;
- les requêtes commerciales ;
- les requêtes documentaires ;
- les requêtes comparatives ;
- les requêtes liées à l’intelligence artificielle ;
- les clusters SEO, GEO et GIO ;
- les pages cibles ;
- les risques de cannibalisation ;
- les variantes multilingues ;
- les règles de suivi et d’évolution.

Cette cartographie n’est pas un relevé de volumes de recherche payant.

Les priorités sont établies à partir de :

- la pertinence pour FlatCMS ;
- l’intention utilisateur ;
- la concurrence visible dans les moteurs ;
- la proximité avec les fonctionnalités réelles ;
- la capacité de FlatCMS à produire une réponse meilleure et vérifiable ;
- la valeur de conversion ;
- le potentiel de citation par les moteurs génératifs.

Les volumes et positions réels devront être complétés après publication avec :

- Google Search Console ;
- Bing Webmaster Tools ;
- Google Trends ;
- Matomo ou l’outil analytics retenu ;
- données de conversion ;
- éventuel outil spécialisé de recherche de mots-clés.

## 2. Principes stratégiques

## 2.1 Trois niveaux de visibilité

### Niveau 1 — Être trouvé par son nom

Objectif :

```text
FlatCMS
FlatCMS France
FlatCMS CMS
FlatCMS PHP
FlatCMS v1.0.0
```

Objectif :

```text
CMS sans base de données
CMS flat-file PHP
CMS PHP JSON
CMS léger
CMS sans MySQL
```

### Niveau 3 — Être trouvé par l’expertise

Objectif :

```text
architecture HMVC PHP
autoloading PSR-4 CMS
stockage JSON CMS
CMS agent-ready
données structurées CMS
```

Les trois niveaux doivent progresser ensemble.

---

## 2.2 Une requête principale, une page cible

Chaque requête stratégique doit disposer d’une page principale clairement identifiée.

Exemple :

```text
CMS sans base de données
→ /fr-FR/comparatifs/cms-sans-base-de-donnees/

Stockage JSON FlatCMS
→ /fr-FR/architecture/stockage-json/

Installer FlatCMS
→ /fr-FR/documentation/installation/

CMS agent-ready
→ /fr-FR/agent-ready/
```

Les autres pages peuvent employer l’expression, mais elles doivent pointer vers la page cible.

---

## 2.3 Pas de bourrage de mots-clés

Les expressions doivent être intégrées naturellement dans :

- le title ;
- le H1 ;
- l’introduction ;
- les H2 pertinents ;
- les ancres de liens ;
- le texte alternatif lorsque descriptif ;
- les métadonnées sociales ;
- les contenus connexes.

La répétition mécanique n’est pas un objectif.

---

## 2.4 Priorité aux contenus utiles

Google indique que les mêmes fondamentaux SEO restent applicables à ses fonctionnalités IA : page indexable, contenu utile et textuel, liens internes, bonne expérience et données structurées cohérentes avec le contenu visible.

Il n’existe pas de balisage spécial obligatoire pour apparaître dans AI Overviews ou AI Mode.

La stratégie GEO/GIO de FlatCMS repose donc sur :

- la qualité factuelle ;
- la clarté ;
- la structure ;
- la cohérence de l’écosystème de pages ;
- la citabilité ;
- l’autorité ;
- l’accessibilité technique.

# 30. Stratégie GEO de mesure

Les moteurs génératifs ne fournissent pas tous des données comparables.

Mesures possibles :

- citations de FlatCMS ;
- liens générés vers le domaine ;
- exactitude des descriptions ;
- requêtes où FlatCMS est mentionné ;
- pages citées ;
- cohérence de la marque ;
- conversions issues des référents IA ;
- mentions sans lien.

## Protocole

Créer une liste fixe de requêtes :

```text
Quel CMS PHP sans base de données choisir ?
Qu’est-ce qu’un CMS flat-file ?
Quelle alternative légère à WordPress ?
Quel CMS utilise JSON au lieu de MySQL ?
Quel CMS PHP est conçu pour les agents IA ?
Comment installer FlatCMS ?
FlatCMS est-il open source ?
```

Tester périodiquement plusieurs moteurs, dater les résultats et conserver les captures ou exports.

Ne pas modifier les contenus uniquement pour une réponse isolée.
