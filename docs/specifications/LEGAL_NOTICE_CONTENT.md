l’architecture et les coordonnées légales doivent faire l’objet d’une
validation particulière.

inutile.

des infrastructures différentes.

Éviter une clause générale imposant systématiquement un tribunal
professionnel à un consommateur.

Les règles de compétence doivent être rédigées dans les CGV selon :

- client professionnel ;
- consommateur ;
- résidence ;
- droit impératif ;
- médiation.

et le public concerné le rendent applicable.

## Copyright du footer

Formulation prudente :

```text
© [ANNÉE] FlatCMS — Certains droits réservés.
```

ou :

```text
© [ANNÉE] Alain BROYE / FlatCMS.
```

Ne pas utiliser automatiquement :

```text
Tous droits réservés
```

pour englober le logiciel open source, les dépendances tierces et les
contenus sous licences distinctes.

doit rester directement accessible et utilisable.

d’assistance.

toutes les langues :

- identité ;
- adresse ;
- registre ;
- capital ;
- TVA ;
- hébergeur ;
- directeur ;
- date.

Une traduction juridique doit être présentée comme telle.

```text
La version française prévaut, sous réserve du droit impératif applicable.
```

conclus avec des consommateurs étrangers.
